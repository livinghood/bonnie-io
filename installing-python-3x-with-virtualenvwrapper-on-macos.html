Installing Python 3.x with virtualenvwrapper on MacOS

// under python3 virtualenv python

For the last few weeks I’ve been playing with Python and reading about Python 3. The best way to get to know Python 3 is to dive into it (I thought). Trying not to break everything, it is a good idea to create separate environments for different Python projects. Why? So that I can use both Python 2 and Python 3!

Install Python 3 with Homebrew:

# Always update brew before installing anything with it
$ brew update

$ brew install python3

In the previous post, I mentioned how to install virtualenv. Virtualenvwrapper is a set of extensions to virtualenv, making it easter to manage multiple projects without conflicts between the installed dependencies.

Create a virtual environment with virtualenvwrapper with Python 3:

$ pip install virtualenvwrapper
$ export WORKON=~/virtualenvs
$ source /usr/local/bin/virtualenvwrapper.sh

$ mkvirtualenv --python=/usr/local/bin/python3 python3env

Great! We have just created a virtualenv named python3env.

By adding the following lines to .bash_profile to change the path to virtualenvwrapper.sh to where pip was installed, you won’t need to type source /usr/local/bin/virtualenvwrapper.sh everytime when you want to use virtualenvwrapper.

export WORKON_HOME=$HOME/virtualenvs
source /usr/local/bin/virtualenvwrapper.sh

We can add another virtualenv and switch between environments with workon:

(python3env)$ mkvirtualenv python2env
(python2env)$ workon python3env
(python3env)$

Check which version Python in use is:

(python3env)$ python -V
Python 3.4.2

To deactivate a virtual environment: 

(python3env)$ deactivate

To remove a virtualenv: (You have to deactivate the virtualenv before removing it)

$ rmvirtualenv python3env

Voilà! Now we can use both Python 2.7.x and Python 3 without destroying the harmonous ecosystem! Time to dig in, install some dependencies and learn more about virtualenv and virtualenvwrapper.
